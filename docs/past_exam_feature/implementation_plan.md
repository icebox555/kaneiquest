# 過去問演習機能 実装計画

## 概要
ユーザーが「過去10年分」などの年度を指定して問題を解けるようにする。
また、実際の過去問データをDBに投入するための基盤を整備する。

## データ構造の変更
### `questions` テーブル
- `exam_year` (int4, nullable): 出題年度 (例: 2024, 38 etc. 第38回なら38、西暦なら2024。統一が必要。**西暦(20xx)** を推奨)
- `question_number` (int4, nullable): その年度の問番号

## UI/UXの変更
### 1. 練習問題選択ページ (`/practice`)
- 現在: カテゴリ一覧のみ
- 変更案:
    - タブ切り替え: 「分野別で解く」 / 「年度別で解く」
    - 「年度別」タブ: 過去の年度リスト (2024年, 2023年...) を表示し、選択するとその年度の全問題(または科目選択)へ。

### 2. Admin管理画面
- 問題作成・編集フォームに `年度` と `問番号` の入力フィールドを追加（任意項目）。

## データインポート戦略
1.  ユーザーからテキスト/PDF/画像を受け取る。
2.  LLMを使用して以下のJSON形式に構造化する。
    ```json
    [
      {
        "year": 2024,
        "number": 1,
        "category_name": "社会・環境と健康",
        "content": "...",
        "options": [...],
        "answer_index": 0,
        "explanation": "..."
      }
    ]
    ```
3.  構造化したデータをDBにインポートするスクリプトを実行。

## 実装ステップ
1.  **DBマイグレーション**: カラム追加。
2.  **Admin UI**: 入力フォーム改修。
3.  **User UI**: `/practice` ページの改修 (とりあえず年度を表示するためのモックデータまたは既存データの更新が必要)。
4.  **データ投入**: 実際のデータを使って検証。

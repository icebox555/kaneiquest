# 決済サービス導入提案

Kanei QuestのようなNext.js + Supabase構成のWebアプリケーションにおいて、推奨される決済サービス比較です。

## 推奨: Stripe

最も推奨される選択肢です。

### メリット
- **開発親和性:** Next.js (React) や Supabase との連携ライブラリが充実しており、実装が標準化されています。
- **機能:** サブスクリプション（定期課金）、単発決済、クーポン、トライアル機能などが標準で備わっています。
- **UI:** 「Stripe Checkout」を使えば、セキュアで綺麗な決済画面を自分で作らずに導入できます。
- **手数料:** 3.6% (国内カード) と標準的。

### デメリット
- グローバルな税務処理（VATなど）は自分で設定が必要（日本国内メインなら問題なし）。

## 次点: Lemon Squeezy

### メリット
- **Merchant of Record (MoR):** 消費税や海外の税務処理を代行してくれるため、税務リスクが低いです。
- **導入:** Stripe同様に簡単です。

### デメリット
- 手数料が少し高い（5% + 50¢ など）。
- 日本語ドキュメントや国内事例がStripeに比べると少なめ。

## その他の選択肢
- **Pay.jp:** 日本発のサービスで、ドキュメントが完全日本語。シンプルですが、機能（特にサブスクリプションの柔軟性）はStripeに劣る場合があります。
- **PayPal:** ユーザーには馴染みがありますが、APIが少し古く、実装体験はStripeの方がモダンです。

## 結論と次のステップ

**Stripe の導入を強く推奨します。**

### Stripe導入のステップ案
1. Stripeアカウント作成（テスト環境はすぐに利用可能）
2. プロダクト（月額プラン等）の作成
3. アプリ内に「アップグレード」ボタンを設置
4. APIルート (`/api/checkout`) の作成
5. Webhook処理（決済完了時にSupabaseのユーザー権限を更新）の実装

この方向でよろしければ、**Stripeの実装計画**を作成します。
